## 📡 MISSION BRIEFING

I started out writing backend code — Java and Python, Spring Boot on the Java side — the kind of work where a broken API or a bad SQL query becomes everyone's problem by lunch. That's where I learned to care about things actually working, not just running once on my machine.

Lately I've been pointing that same instinct at AI systems. Most of what's on this profile right now is personal projects, not production deployments but I try to hold them to the same bar: not "does it look good in a demo," but "can I actually measure that it works." My RAG project, for example, isn't just a chatbot I ran an evaluation comparing hallucination rates with and without retrieval, on real CVEs, and wrote down the numbers instead of just eyeballing it.

**What I'm doing right now:**
- 🤖 Building small, working AI systems — RAG pipelines, MCP servers, retrieval over real data
- 🔍 Poking at how LLMs actually behave, where they hallucinate, and how to catch it
- ⚙️ Building backend services (Java/Spring Boot, Python/FastAPI) that AI features can sit on top of without falling over
- 🐛 Fixing the unglamorous stuff — broken DB connections, flaky auth, the bugs that block everything else

I like projects I can back up with evidence, not just a screenshot.
